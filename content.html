Two common types of files are plain text files and structured files. These have different characteristics and are used for different purposes.

Plain Text Files

Plain text files contain text characters without rich formatting (for example, embedded fonts, colours, or layout metadata). They are simple and widely compatible, making them ideal for storing human-readable information. Examples include .txt files and source code files.

Structured Files

Structured files contain data that is organised in a defined format, often with specific fields and records. They are used to store complex information that can be processed consistently by software. Examples include .csv files, .json files, and .xml files (used in many modern Microsoft Office file formats).

In many cases, programs that open structured files interpret structural markers to display the data in a user-friendly way, often without showing those markers directly. As a result, when viewing structured files, you may not see all of the raw content exactly as it is stored.
